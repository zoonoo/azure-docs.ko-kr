# <a name="interpreting-your-scorecard"></a>성과 기록표 해석

성과 기록표 탭에는 테스트에서 집계 되 고 분석 된 결과가 포함 됩니다. 각 테스트에는 고유한 성과 기록표가 있습니다. 성과 기록표는 네트워크 요구 사항에 대 한 데이터 기반 결과를 제공 하는 측정 결과의 빠르고 의미 있는 요약을 제공 합니다. 인터넷 분석기는 분석을 처리 하므로 결정에 집중할 수 있습니다.

성과 기록표 탭은 인터넷 분석기 리소스 메뉴에서 찾을 수 있습니다. 


## <a name="filters"></a>필터

* ***테스트:*** 결과를 보려는 테스트를 선택 합니다. 각 테스트에는 고유한 성과 기록표가 있습니다. 분석을 완료 하기에 충분 한 데이터가 있는 경우 테스트 데이터가 표시 됩니다. 대부분의 경우 24 시간 이내 여야 합니다. 
* 기간 ***& 종료 날짜:*** 3 개의 성과 기록표가 매일 생성 됩니다. 각 성과 기록표는 각각 24 시간 이전 (일), 7 일 전 (주) 및 30 일 (월)의 집계 기간을 반영 합니다. "종료 날짜" 필터를 사용 하 여 보려는 기간의 마지막 날짜를 선택 합니다. 
* ***국가:*** 최종 사용자가 있는 각 국가에 대해 성과 기록표가 생성 됩니다. 전역 필터에는 모든 최종 사용자가 포함 됩니다.

## <a name="measurement-count"></a>측정 수

측정 수는 분석의 신뢰도에 영향을 줍니다. 개수가 높을수록 결과가 더 정확해 집니다. 최소 테스트는 일별 끝점 당 최소 100 측정치를 목표로 해야 합니다. 측정 횟수가 너무 낮으면 응용 프로그램에서 JavaScript 클라이언트가 더 자주 실행 되도록 구성 하세요. 끝점 A와 B의 측정 수는 작은 차이를 기대 하 고 확인 하는 것은 매우 유사 해야 합니다. 큰 차이의 경우 결과를 신뢰할 수 없습니다.

## <a name="percentiles"></a>백분위수

밀리초 단위로 측정 되는 대기 시간은 인터넷에서 원본 및 대상 간의 속도를 측정 하는 데 널리 사용 되는 메트릭입니다. 산술 평균과 같은 통계를 사용 하는 경우 결과를 기울이는 긴 대기 시간 값의 "긴 비상"이 있으므로 대기 시간 데이터는 정상적으로 분산 되지 않습니다 (즉, "종 곡선"은 따르지 않음). 또는 백분위 수를 사용 하 여 데이터를 분석할 수 있습니다. 예를 들어 중앙값 또는 50 번째 백분위 수는 분포의 중간을 요약 합니다. 값의 절반은 위쪽이 고 나머지는 아래입니다. 75 번째 백분위 수 값은 분포에 있는 모든 값의 75% 보다 큼을 의미 합니다. 인터넷 분석기는 P50, P75 및 P95으로 백분위 수의 약어를 나타냅니다.

Internet Analyzer 백분위 수는 _샘플 메트릭입니다_. 이는 진정한 _모집단 메트릭과_대조 됩니다. 예를 들어 주간 진정한 인구는 남부 캘리포니아 대학에 있는 학생 간의 대기 시간을 단축 하 고, Microsoft는 해당 일 동안의 모든 요청에 대 한 평균 대기 시간 값입니다. 실제로 모든 요청 값을 측정 하는 것은 실용적이 지 않으므로 합리적인 크기의 샘플이 진정한 모집단을 대표 한다고 가정 합니다.

분석을 위해 P50 (중앙값)는 대기 시간 분포의 예상 값으로 유용 합니다. P95와 같은 더 높은 백분위 수는 최악의 경우 대기 시간을 확인 하는 데 유용 합니다. 일반적으로 고객의 대기 시간을 이해 하는 데 관심이 있는 경우 P50는 초점을 맞출 올바른 메트릭입니다. 최악의 경우의 고객에 대 한 성능을 이해 하는 데 관심이 있는 경우 P95는 포커스 여야 합니다. P75는 이러한 두 가지 간에 균형을 유지 합니다.


## <a name="deltas"></a>델타

델타는 끝점 A와 B에 대 한 메트릭 값의 차이입니다. 델타는 A에 대 한 B의 혜택을 보여 주기 위해 계산 됩니다. 양수 값은 A 보다 더 나은 B를 나타냅니다. 반면 음수 값은 B의 성능이 저하 됨을 나타냅니다. 델타는 절대 경로 (예: 10 밀리초) 또는 상대 (5%) 일 수 있습니다.

## <a name="confidence-interval"></a>예측 

신뢰 구간 (CI)은 중앙값, P75 또는 평균과 같은 모집단 메트릭을 포함 하는 확률이 있는 값의 범위입니다. 95% CI를 사용 하는 일반적인 통계 규칙을 따릅니다.

인터넷 분석기의 경우 샘플 메트릭이 실제 모집단 메트릭과 매우 근접 한 것으로 표시 되기 때문에 좁은 신뢰 간격이 양호 합니다. 와이드 신뢰 간격이 있으면 샘플 메트릭이 진정한 모집단 메트릭을 반영 한다는 확신도가 낮아집니다. CI를 개선 하는 가장 좋은 방법은 측정 횟수를 늘리는 것입니다.

## <a name="time-series"></a>시계열 

시계열은 시간이 지남에 따라 메트릭이 변경 되는 방식을 보여 줍니다. 인터넷에는 최고 트래픽 기간, 평일-주말 모집단 차이, 휴일 등의 성능에 영향을 주는 많은 임시 요소가 있습니다.

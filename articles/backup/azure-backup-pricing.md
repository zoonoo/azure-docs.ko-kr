# <a name="azure-backup-pricing"></a>Azure Backup 가격 책정

Azure Backup 가격 책정에 대해 알아보려면 [Azure Backup 가격 페이지](https://azure.microsoft.com/pricing/details/backup/)를 참조하세요.

## <a name="download-detailed-estimates-for-azure-backup-pricing"></a>Azure Backup 가격 책정에 대한 자세한 예상 비용 다운로드

예산을 수립하거나 비용을 비교하기 위해 비용을 추정하려면 자세한 [Azure Backup 가격 책정 예측 도구](https://aka.ms/AzureBackupCostEstimates)를 다운로드합니다.  

### <a name="what-does-the-estimator-contain"></a>예측 도구에는 무엇이 포함되나요?

Azure Backup 비용 예측 도구 시트에는 Azure Backup을 사용하여 백업하려는 모든 가능한 워크로드를 추정할 수 있는 옵션이 있습니다. 이러한 워크로드는 다음과 같습니다.

- Azure VM
- 온-프레미스 서버
- Azure VM의 SQL
- Azure VM의 SAP HANA
- Azure 파일 공유

## <a name="estimate-costs-for-backing-up-azure-vms-or-on-premises-servers"></a>Azure VM 또는 온-프레미스 서버 백업에 대한 비용 예측

Azure Backup을 사용하여 Azure VM 또는 온-프레미스 서버를 백업하는 데 드는 비용을 예측하려면 다음 매개 변수가 필요합니다.

- 백업하려는 VM 또는 온사이트 서버의 크기
  - 백업하는 데 필요한 디스크 또는 서버의 "사용되는 크기"를 입력합니다.

- 해당 크기의 서버 수

- 이러한 서버에서 예상되는 데이터 변동량<br>
  변동은 데이터의 변경량을 나타냅니다. 예를 들어 백업할 데이터가 200GB이고 그 중 10GB가 매일 변경되는 VM이 있는 경우 일일 변동률은 5%입니다.

  - 변동률이 높을수록 더 많은 데이터를 백업하게 됩니다.

  - 파일 서버의 경우 **낮음** 또는 **보통** 을 선택하고, 데이터베이스를 실행하는 경우 **높음** 을 선택합니다.

  - **변동(%)** 을 알고 있는 경우 **사용자 입력(%)** 옵션을 사용할 수 있습니다.

- 백업 정책 선택

  - "매일" 백업을 유지하는 데 예상하는 기간은 얼마인가요? (일)

  - "매주" 백업을 유지하는 데 예상하는 기간은 얼마인가요? (주)

  - "매월" 백업을 유지하는 데 예상하는 기간은 얼마인가요? (개월)

  - "매년" 백업을 유지하는 데 예상하는 기간은 얼마인가요? (년)

  - "즉시 복원 스냅샷" 백업을 유지하는 데 예상하는 기간은 얼마인가요? (1-5일)

    - 이 옵션을 사용하면 디스크에 저장된 스냅샷을 사용하여 7일 전부터 빠르게 복원할 수 있습니다.

- **선택 사항** – 선택적 디스크 백업

  - Azure VM을 백업하는 동안 **선택적 디스크 백업** 옵션을 사용하는 경우, **디스크 제외** 옵션을 선택하고 크기 측면에서 백업에서 제외되는 디스크의 비율을 입력합니다. 예를 들어 각 디스크에서 200GB가 사용되는 세 개의 디스크에 VM이 연결되어 있고 그 중 두 개를 백업에서 제외하려면 66.7%를 입력합니다.

- **선택 사항** – 백업 스토리지 중복

  - 이는 백업 데이터가 전달되는 스토리지 계정의 중복을 나타냅니다. 최고의 가용성을 얻으려면 **GRS** 를 사용하는 것이 좋습니다. 백업 데이터의 복사본이 다른 지역에 유지되도록 하므로 여러 규정 준수 표준을 충족하는 데 도움이 됩니다. 엔터프라이즈 수준 백업이 필요하지 않은 개발 또는 테스트 환경을 백업하는 경우 중복을 **LRS** 로 변경합니다. [지역 간 복원](backup-azure-arm-restore-vms.md#cross-region-restore)을 백업에 사용하도록 설정하는 경우 비용을 파악하려면 시트에서 **RAGRS** 옵션을 선택합니다.

- **선택 사항** – 지역별 가격 책정 수정 또는 할인율 적용

  - 다른 지역 또는 할인율에 대한 예상 비용을 확인하려면 **다른 지역에 대한 비용을 예측하시겠습니까?** 옵션에 대해 **예** 를 선택하고 예측을 실행할 비율을 입력합니다.

## <a name="estimate-costs-for-backing-up-sql-servers-in-azure-vms"></a>Azure VM의 SQL 서버 백업에 대한 비용 예측

Azure Backup을 사용하여 Azure VM에서 실행되는 SQL 서버를 백업하는 데 드는 비용을 예측하려면 다음 매개 변수가 필요합니다.

- 백업하려는 SQL 서버의 크기

- 위 크기의 SQL 서버 수

- SQL 서버의 백업 데이터에 대해 예상되는 압축

  - 대부분의 Azure Backup 고객은 SQL 압축을 **사용하도록 설정** 된 경우 백업 데이터가 SQL 서버 크기와 비교하여 80%의 비율로 압축되었음을 알 수 있습니다.

  - 다른 압축이 예상되는 경우 이 필드에서 숫자를 입력합니다.

- 예상되는 로그 백업 크기

  - %는 일일 로그 크기를 SQL 서버 크기에 대한 %로 나타냅니다.

- 이러한 서버에서 예상되는 일일 데이터 변동량

  - 일반적으로 데이터베이스의 변동은 "높음"입니다.

  - **변동(%)** 을 알고 있는 경우 **사용자 입력(%)** 옵션을 사용할 수 있습니다.

- 백업 정책 선택

  - 백업 유형

    - 가장 효과적으로 선택할 수 있는 정책은 매일/매월/매년 전체 백업이 포함된 **매일 차등** 입니다. Azure Backup은 한 번의 클릭으로 차등 백업에서 복원할 수도 있습니다.

    - 매일/매주/매월/매년 전체 백업이 포함된 정책을 사용하도록 선택할 수도 있습니다. 이 옵션은 첫 번째 옵션보다 약간 더 많은 스토리지를 사용합니다.

  - "로그" 백업을 유지하는 데 예상하는 기간은 얼마인가요? (일)[7-35]

  - "매일" 백업을 유지하는 데 예상하는 기간은 얼마인가요? (일)

  - "매주" 백업을 유지하는 데 예상하는 기간은 얼마인가요? (주)

  - "매월" 백업을 유지하는 데 예상하는 기간은 얼마인가요? (개월)

  - "매년" 백업을 유지하는 데 예상하는 기간은 얼마인가요? (년)

- **선택 사항** – 백업 스토리지 중복

  - 이는 백업 데이터가 전달되는 스토리지 계정의 중복을 나타냅니다. 최고의 가용성을 얻으려면 **GRS** 를 사용하는 것이 좋습니다. 백업 데이터의 복사본이 다른 지역에 유지되도록 하므로 여러 규정 준수 표준을 충족하는 데 도움이 됩니다. 엔터프라이즈 수준 백업이 필요하지 않은 개발 또는 테스트 환경을 백업하는 경우 중복을 **LRS** 로 변경합니다.

- **선택 사항** – 지역별 가격 책정 수정 또는 할인율 적용

  - 다른 지역 또는 할인율에 대한 예상 비용을 확인하려면 **다른 지역에 대한 비용을 예측하시겠습니까?** 옵션에 대해 **예** 를 선택하고 예측을 실행할 비율을 입력합니다.

## <a name="estimate-costs-for-backing-up-azure-file-shares"></a>Azure 파일 공유 백업에 대한 비용 예측

Azure Backup에서 제공하는 [스냅샷 기반 백업 솔루션](azure-file-share-backup-overview.md)을 사용하여 Azure 파일 공유를 백업하는 데 드는 비용을 예측하려면 다음 매개 변수가 필요합니다.

- 백업하려는 파일 공유의 크기( **GB** ).

- 여러 스토리지 계정에 분산된 파일 공유를 백업하려면 파일 공유를 위의 크기로 호스팅하는 스토리지 계정의 수를 지정합니다.

- 백업하려는 파일 공유에서 예상되는 데이터 변동량. <br>변동은 데이터의 변경량을 나타내며 스냅샷 스토리지 크기에 직접적인 영향을 줍니다. 예를 들어 백업할 데이터가 200GB이고 그 중 10GB가 매일 변경되는 파일 공유가 있는 경우 일일 변동률은 5%입니다.
  - 변동률이 높을수록 파일 공유 콘텐츠의 매일 데이터 변경량이 많으므로 증분 스냅샷(데이터 변경만 캡처) 크기도 더 커집니다.
  - 파일 공유 특성 및 사용량에 따라 낮음(1%), 보통(3%) 또는 높음(5%)을 선택합니다.
  - 파일 공유에 대한 정확한 **변동(%)** 을 알고 있는 경우 드롭다운에서 **사용자 입력(%)** 옵션을 선택할 수 있습니다. 매일, 매주, 매월 및 매년 변동에 대한 값(%)을 지정합니다.

- 백업된 파일 공유를 호스팅하는 스토리지 계정의 스토리지 계정 유형(표준 또는 프리미엄) 및 스토리지 중복 설정. <br>Azure 파일 공유에 대한 현재 백업 솔루션에서 스냅샷은 백업된 파일 공유와 동일한 스토리지 계정에 저장됩니다. 따라서 스냅샷과 관련된 스토리지 비용은 백업된 파일 공유 및 스냅샷을 호스팅하는 스토리지 계정의 계정 유형과 중복 설정에 대한 스냅샷 가격 책정에 따라 Azure 파일 청구서의 일부로 청구됩니다.

- 다른 백업에 대한 보존
  - "매일" 백업을 유지하는 데 예상하는 기간은 얼마인가요? (일)
  - "매주" 백업을 유지하는 데 예상하는 기간은 얼마인가요? (주)
  - "매월" 백업을 유지하는 데 예상하는 기간은 얼마인가요? (개월)
  - "매년" 백업을 유지하는 데 예상하는 기간은 얼마인가요? (년)

  각 범주에서 지원되는 최대 보존 값은 [Azure 파일 공유 지원 매트릭스](azure-file-share-support-matrix.md#retention-limits)를 참조하세요.

- **선택 사항** – 지역별 가격 책정 수정 또는 할인율 적용
  - 예측 도구에서 GB당 스냅샷 스토리지 비용 및 보호된 인스턴스 비용에 대해 설정된 기본값은 미국 동부 지역에 적용되는 값입니다. 다른 지역 또는 할인율에 대한 예상 비용을 확인하려면 **다른 지역에 대한 비용을 예측하시겠습니까?** 옵션에 대해 **예** 를 선택하고 예측을 실행할 비율을 입력합니다.

# <a name="configure-a-managed-application-plan"></a>관리 되는 응용 프로그램 계획 구성

이 문서는 Azure 응용 프로그램 제품에 대 한 관리 되는 응용 프로그램 계획에만 적용 됩니다. 솔루션 템플릿 계획을 구성 하는 경우 [솔루션 템플릿 계획 구성](create-new-azure-apps-offer-solution.md)으로 이동 합니다.

## <a name="define-markets-pricing-and-availability"></a>시장, 가격 책정 및 가용성 정의

모든 플랜은 하나 이상의 시장에서 사용할 수 있어야 합니다. **가격 책정 및 가용성** 탭에서이 요금제를 사용할 수 있는 시장, 가격 및 계획을 모든 사용자에 게 표시할지 또는 특정 고객 에게만 표시할지 여부를 구성할 수 있습니다 (비공개 요금제 라고도 함).

1. **시장** 에서 **시장 편집** 링크를 선택 합니다.
1. 표시 되는 대화 상자에서 요금제를 사용할 수 있도록 할 시장 위치를 선택 합니다. 최소 1 자에서 최대 141 시장을 선택 해야 합니다.

    > [!NOTE]
    > 이 대화 상자에는 Microsoft에서 판매 하 고 사용자를 대신 하 여 세금을 사용 하는 "세금 송금" 국가만 필터링 하는 옵션 및 검색 상자가 포함 되어 있습니다.

1. **저장** 을 선택 하 여 대화 상자를 닫습니다.

## <a name="define-pricing"></a>가격 책정 정의

**가격** 상자에이 요금제에 대 한 월별 가격을 제공 합니다. 이 가격은이 솔루션에서 배포한 리소스에 의해 발생 하는 Azure 인프라 또는 사용량 기반 비용에 추가 됩니다.

또한 월별 가격 외에 [요금 청구](partner-center-portal/azure-app-metered-billing.md)를 사용하여 비표준 단위의 사용량에 해당하는 가격을 설정할 수 있습니다. 월별 가격을 0으로 설정하고 원하는 경우 요금 청구를 사용하여 전용 요금을 청구할 수 있습니다.

가격이 USD (USD = 미국 달러)로 설정 된 경우에는 저장할 때 현재 환율을 사용 하 여 선택한 모든 시장의 현지 통화로 변환 됩니다. 가격 책정 스프레드시트를 내보내고 각 시장에서 가격을 검토하여 게시하기 전에 해당 가격의 유효성을 검사합니다. 개별 시장에서 사용자 지정 가격을 설정하려면 가격 책정 스프레드시트를 수정하고 가져옵니다.

### <a name="add-a-custom-meter-dimension-optional"></a>사용자 지정 측정기 차원 추가 (선택 사항)

1. **Marketplace 계량 서비스 차원** 에서 **사용자 지정 측정기 차원 추가 (최대 18 개)** 링크를 선택 합니다.
1. **ID** 상자에 사용 이벤트를 내보내는 동안 변경할 수 없는 식별자 참조를 입력 합니다.
1. **표시 이름** 상자에 차원과 연결 된 표시 이름을 입력 합니다. 예를 들어 "보낸 문자 메시지"가 있습니다.
1. **측정 단위** 상자에 청구 단위에 대 한 설명을 입력 합니다. 예를 들어 "문자 단위 메시지" 또는 "100 전자 메일"을 사용할 수 있습니다.
1. 단가 **별 가격** 상자에 차원의 한 단위에 대 한 가격을 입력 합니다.
1. **기준에 포함 된 월간 수량** 상자에 월간 요금을 지불 하는 고객을 위해 매월 포함 되는 차원의 수량 (정수)을 입력 합니다. 무제한 수량을 설정 하려면 대신 확인란을 선택 합니다.
1. 다른 사용자 지정 측정기 차원을 추가 하려면 1 ~ 7 단계를 반복 합니다.

### <a name="set-custom-prices-optional"></a>사용자 지정 가격 설정 (선택 사항)

USD(미국 달러)로 설정된 가격은 저장할 때 현재 환율을 사용하여 선택한 모든 시장의 현지 통화로 변환됩니다. 가격 책정 스프레드시트를 내보내고 각 시장에서 가격을 검토하여 게시하기 전에 해당 가격의 유효성을 검사합니다. 개별 시장에서 사용자 지정 가격을 설정하려면 가격 책정 스프레드시트를 수정하고 가져옵니다.

플랜이 게시된 후에 변경 가능한 내용에는 일부 제한이 있으므로 게시하기 전에 가격을 신중하게 검토해야 합니다.

> [!NOTE]
> 플랜의 시장 가격이 게시된 후에는 나중에 변경할 수 없습니다.

개별 시장에서 사용자 지정 가격을 설정 하려면 가격 책정 스프레드시트를 내보내고 수정한 후 가져옵니다. 이 가격의 유효성을 검사 하 고 이러한 설정을 소유 하 게 됩니다. 자세한 내용은 [사용자 지정 가격](plans-pricing.md#custom-prices)을 참조 하세요.

1. 가격 책정 데이터를 내보낼 수 있도록 먼저 가격 책정 변경 내용을 저장해야 합니다. **가격 책정 및 가용성** 탭의 아래쪽에 있는 **초안 저장** 을 선택 합니다.
1. **가격 책정** 에서 **가격 데이터 내보내기** 링크를 선택 합니다.
1. Microsoft Excel에서 exportedPrice.xlsx 파일을 엽니다.
1. 스프레드시트에서 가격 정보에 대 한 업데이트를 수행 하 고 파일을 저장 합니다.

   Excel에서 편집을 사용 하도록 설정 해야 파일을 업데이트할 수 있습니다.

1. 가격 책정 **및 가용성** 탭의 **가격 책정** 에서 **가격 책정 데이터 가져오기** 링크를 선택 합니다.
1. 표시 되는 대화 상자에서 **예** 를 클릭 합니다.
1. 업데이트 한 exportedPrice.xlsx 파일을 선택한 다음 **열기** 를 클릭 합니다.

## <a name="choose-who-can-see-your-plan"></a>요금제를 볼 수 있는 사용자 선택

모든 사용자 또는 특정 대상에 게 표시 되도록 각 계획을 구성할 수 있습니다. 사용자가 할당 한 각 구독 ID에 대 한 설명을 포함 하는 옵션과 함께 Azure 구독 Id를 사용 하 여 개인 대상 그룹에 대 한 액세스 권한을 부여 합니다. 을 사용 하 여 최대 10 개의 구독 Id를 수동으로 또는 최대 1만 구독 Id를 추가할 수 있습니다. CSV 파일. Azure 구독 ID는 GUID로 표시되며 문자는 소문자여야 합니다.

> [!NOTE]
> 비공개 요금제를 게시 하는 경우 나중에 공용으로 표시를 변경할 수 있습니다. 그러나 공개 계획을 게시 한 후에는 해당 표시 유형을 개인으로 변경할 수 없습니다.

**계획 표시 유형** 아래에서 다음 중 하나를 수행 합니다.

- 계획을 공개로 설정 하려면 _라디오 단추_ 라고도 하는 **공용** 옵션 단추를 선택 합니다.
- 계획을 비공개로 설정 하려면 **개인** 옵션 단추를 선택한 다음 Azure 구독 id를 수동으로 또는 CSV 파일에 추가 합니다.

> [!NOTE]
> 프라이빗 또는 제한된 대상 그룹은 **미리 보기** 탭에서 정의한 미리 보기 대상 그룹과 다릅니다. 미리 보기 대상 그룹은 제품이 마켓플레이스에 라이브로 게시되기 ‘전에’ 해당 제품에 액세스할 수 있습니다. 프라이빗 대상 그룹 선택은 특정 플랜에만 적용되지만, 미리 보기 대상 그룹은 유효성 검사를 위해 모든 플랜(프라이빗 및 퍼블릭)을 볼 수 있습니다.

## <a name="define-the-technical-configuration"></a>기술 구성 정의

**기술 구성** 탭에서 고객이 계획을 배포 하 고 패키지에 대 한 버전 번호를 제공할 수 있는 배포 패키지를 업로드 합니다. 또한 다른 기술 정보도 제공 합니다.

> [!NOTE]
> **계획 설정** 탭에서 다른 계획의 패키지를 다시 사용 하도록 선택한 경우에는이 탭이 표시 되지 않습니다. 그렇다면 [계획 보기](#view-your-plans)로 이동 합니다.

### <a name="assign-a-version-number-for-the-package"></a>패키지에 대 한 버전 번호를 할당 합니다.

**버전** 상자에서 기술 구성의 현재 버전을 제공 합니다. 이 페이지에 변경 내용을 게시할 때마다 이 버전을 증분합니다. 버전 번호는 integer 형식 이어야 합니다. 정수. 정수. 예를 들어 `1.0.2`.

### <a name="upload-a-package-file"></a>패키지 파일 업로드

**패키지 파일 (.zip)** 에서 패키지 파일을 회색 상자로 끌거나 **파일 찾아보기** 링크를 선택 합니다.

> [!NOTE]
> 파일을 업로드하는 동안 문제가 발생한 경우 로컬 네트워크가 파트너 센터에서 사용하는 `https://upload.xboxlive.com` 서비스를 차단하지 않는지 확인하세요.

#### <a name="previously-published-packages"></a>이전에 게시된 패키지

**이전에 게시된 패키지** 하위 탭에서는 기술 구성의 게시된 모든 버전을 볼 수 있습니다.

### <a name="enable-just-in-time-jit-access-optional"></a>JIT (just-in-time) 액세스 사용 (선택 사항)

이 계획에 대 한 JIT 액세스를 사용 하도록 설정 하려면 **jit (just-in-time) 액세스 사용** 확인란을 선택 합니다. 관리형 애플리케이션의 소비자가 사용자 계정에 영구 액세스 권한을 부여하도록 요구하려면 이 옵션의 선택을 취소합니다. 이 옵션에 대해 자세히 알아보려면 [JIT (just-in-time) 액세스](plan-azure-app-managed-app.md#just-in-time-jit-access)를 참조 하세요.

### <a name="select-a-deployment-mode"></a>배포 모드 선택

**전체** 또는 **증분** 배포 모드를 선택 합니다.

- **완료** 모드에서 고객이 응용 프로그램을 다시 배포 하면 리소스가의 [mainTemplate.js](/azure/azure-resource-manager/managed-applications/publish-service-catalog-app?tabs=azure-powershell#create-the-arm-template.md)에 정의 되어 있지 않은 경우 관리 되는 리소스 그룹의 리소스가 제거 됩니다.
- **증분** 모드에서 응용 프로그램을 다시 배포 하면 기존 리소스는 변경 되지 않습니다.

배포 모드에 관한 자세한 내용은 [Azure Resource Manager 배포 모드](/azure/azure-resource-manager/deployment-modes.md)를 참조하세요.

### <a name="provide-a-notification-endpoint-url"></a>알림 끝점 URL 제공

**알림 끝점 URL** 상자에이 계획 버전의 관리 되는 응용 프로그램 인스턴스에 대 한 모든 CRUD 작업에 대 한 알림을 수신 하는 HTTPS Webhook 끝점을 제공 합니다.

### <a name="customize-allowed-customer-actions-optional"></a>허용 된 고객 작업 사용자 지정 (선택 사항)

1. 고객이 관리 되는 리소스에 대해 수행할 수 있는 작업을 지정 하 고 `*/read` , 기본적으로 사용할 수 있는 "" 작업을 지정 하려면 **허용 된 고객 작업 사용자 지정** 상자를 선택 합니다.
1. 표시 되는 상자에서 고객이 수행할 수 있도록 하는 추가 제어 작업 및 허용 되는 데이터 작업을 세미콜론으로 구분 하 여 제공 합니다. 예를 들어 소비자가 가상 컴퓨터를 다시 시작 하도록 허용 하려면 `Microsoft.Compute/virtualMachines/restart/action` **허용 된 제어 작업** 상자에를 추가 합니다.

### <a name="choose-who-can-manage-the-application"></a>응용 프로그램을 관리할 수 있는 사용자 선택

선택한 각 Azure 지역에서이 관리 되는 응용 프로그램에 대 한 관리 액세스 권한이 있는 사용자를 지정 합니다. _글로벌 azure_ 및 _Azure Government 클라우드_. Azure AD id를 사용 하 여 관리 되는 리소스 그룹에 권한을 부여 하려는 사용자, 그룹 또는 응용 프로그램을 식별 합니다. 자세한 내용은 [Azure 애플리케이션 제품에 대 한 Azure 관리 응용 프로그램 계획](plan-azure-application-offer.md)을 참조 하세요.

해당 하는 경우 글로벌 Azure 및 Azure Government 클라우드에 대해 다음 단계를 완료 합니다.

1. **Azure Active Directory 테 넌 트 id** 상자에 사용 권한을 부여 하려는 사용자, 그룹 또는 응용 프로그램의 id를 포함 하는 Azure AD 테 넌 트 id (디렉터리 ID 라고도 함)를 입력 합니다.
1. **보안 주체 id** 상자에서 관리 되는 리소스 그룹에 대 한 사용 권한을 부여 하려는 사용자, 그룹 또는 응용 프로그램의 Azure AD 개체 ID를 제공 합니다. 사용자를 보안 주체 ID로 식별 합니다 .이 ID는 Azure Portal의 [사용자 Azure Active Directory 블레이드에서](https://portal.azure.com/#blade/Microsoft_AAD_IAM/UsersManagementMenuBlade/AllUsers) 찾을 수 있습니다.
1. **역할 정의** 목록에서 Azure AD 기본 제공 역할을 선택 합니다. 선택한 역할은 사용자가 고객 구독의 리소스에 대해 가질 수 있는 권한을 설명 합니다.
1. 다른 권한 부여를 추가 하려면 **권한 부여 추가 (최대 100)** 링크를 선택 하 고 1 ~ 3 단계를 반복 합니다.

### <a name="policy-settings-optional"></a>정책 설정 (선택 사항)

최대 5개 정책 및 각 정책 옵션의 인스턴스 하나만 구성할 수 있습니다. 일부 정책에는 추가 매개 변수가 필요합니다.

1. **정책 설정** 에서 **+ 정책 추가 (최대 5)** 링크를 선택 합니다.
1. **이름** 상자에 정책 할당 이름 (50 자로 제한 됨)을 입력 합니다.
1. **정책** 목록 상자에서 고객 구독에서 관리 되는 응용 프로그램에 의해 생성 된 리소스에 적용 될 Azure 정책을 선택 합니다.
1. **정책 매개 변수** 상자에서 감사 및 진단 설정 정책을 적용 해야 하는 매개 변수를 제공 합니다.
1. **정책 sku** 목록 상자에서 정책 sku 유형을 선택 합니다.

    > [!NOTE]
    > 감사 정책에는 _표준 정책_ SKU가 필요 합니다.

## <a name="view-your-plans"></a>요금제 보기

- **초안 저장** 을 선택 하 고 페이지의 왼쪽 **위에서 계획 개요 페이지로 돌아가려면** **계획 개요** 를 선택 합니다.

하나 이상의 계획을 만든 후 계획 **개요** 탭에서 계획 이름, 계획 ID, 계획 유형, 가용성 (공용 또는 개인), 현재 게시 상태 및 사용 가능한 모든 작업을 볼 수 있습니다.

**계획 개요** 탭의 **작업** 열에서 사용할 수 있는 작업은 계획의 상태에 따라 달라 지 며 다음을 포함할 수 있습니다.

- 계획 상태가 **draft** 인 경우 **작업** 열의 링크는 **초안 삭제** 라고 표시 됩니다.
- 계획 상태가 **Live** 인 경우 **작업** 열의 링크는 **판매 계획을 중지** 하거나 **개인 사용자를 동기화** 합니다. **개인 대상 그룹 동기화** 링크는 제안에 대 한 다른 업데이트를 게시 하지 않고 개인 사용자에 대 한 변경 내용만 게시 합니다.
- 이 제품에 대 한 다른 계획을 만들려면 **계획 개요** 탭의 맨 위에서 **+ 새 계획 만들기** 를 선택 합니다. 그런 다음 [Azure 응용 프로그램 제품에 대 한 계획을 만드는 방법](create-new-azure-apps-offer-plans.md)의 단계를 반복 합니다. 그렇지 않으면 계획 만들기를 완료 한 경우 다음 섹션인 다음 단계로 이동 합니다.

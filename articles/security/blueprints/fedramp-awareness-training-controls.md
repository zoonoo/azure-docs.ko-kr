# <a name="awareness-and-training-at"></a>AT(인식 및 교육)

> [!NOTE]
> 이러한 제어는 NIST와 미국 상무부에 의해 NIST 특별 출판물 800-53 개정 4의 일부로 정의됩니다. 각 제어에 대한 테스트 절차 및 지침에 대한 정보는 NIST 800-53 개정 4를 참조하세요.

## <a name="nist-800-53-control-at-1"></a>NIST 800-53 제어 AT-1

#### <a name="security-awareness-and-training-policy-and-procedures"></a>보안 인식과 교육 정책 및 절차

**AT-1** 조직은 목적, 범위, 역할, 책임, 관리 약속, 조직 엔터티 간 조정 및 준수를 다루는 보안 인식 및 교육 정책과 보안 인식 및 교육 정책과 관련된 보안 인식 및 교육 제어의 구현을 용이하게 하는 절차를 개발, 문서화하여 [할당: 조직이 정의한 담당자 또는 역할]에게 배포하고, [할당: 조직이 정의한 빈도]마다 현재 보안 인식 및 교육 정책을 검토하고 업데이트하며 [할당: 조직이 정의한 빈도]마다 보안 인식 및 교육 절차를 검토하고 업데이트합니다.

**책임:** `Customer Only`

|||
|---|---|
| **고객** | 고객의 엔터프라이즈 수준 보안 인식 및 교육 정책과 절차로 이 제어를 위해 충분히 해결할 수 있습니다. |
| **공급자(Microsoft Azure)** | 해당 없음 |


 ## <a name="nist-800-53-control-at-2a"></a>NIST 800-53 제어 AT-2.a

#### <a name="security-awareness-training"></a>보안 인식 교육

**AT-2.a** 조직은 새 사용자에 대한 초기 교육의 일부로 정보 시스템 사용자(관리자, 선임 임원 및 계약자 포함)에게 기본적인 보안 인식 교육을 제공합니다.

**책임:** `Customer Only`

|||
|---|---|
| **고객** | 고객의 엔터프라이즈 수준 보안 인식 교육 프로그램이 모든 시스템 사용자에게 초기 교육을 제공할 수 있습니다. |
| **공급자(Microsoft Azure)** | 해당 없음 |


 ## <a name="nist-800-53-control-at-2b"></a>NIST 800-53 제어 AT-2.b

#### <a name="security-awareness-training"></a>보안 인식 교육

**AT-2.b** 조직은 정보 시스템 변경으로 인해 필요할 때 정보 시스템 사용자(관리자, 선임 임원 및 계약자 포함)에게 기본적인 보안 인식 교육을 제공합니다.

**책임:** `Customer Only`

|||
|---|---|
| **고객** | 고객의 엔터프라이즈 수준 보안 인식 교육 프로그램이 모든 시스템 사용자에게 필요 기반 교육을 제공할 수 있습니다. |
| **공급자(Microsoft Azure)** | 해당 없음 |


 ## <a name="nist-800-53-control-at-2c"></a>NIST 800-53 제어 AT-2.c

#### <a name="security-awareness-training"></a>보안 인식 교육

**AT-2.c** 조직은 [할당: 조직이 정의한 빈도]에 따라 정보 시스템 사용자(관리자, 선임 임원 및 계약자 포함)에게 기본적인 보안 인식 교육을 제공합니다.

**책임:** `Customer Only`

|||
|---|---|
| **고객** | 고객의 엔터프라이즈 수준 보안 인식 교육 프로그램이 모든 시스템 사용자에게 지속적인 교육을 제공할 수 있습니다. |
| **공급자(Microsoft Azure)** | 해당 없음 |


 ### <a name="nist-800-53-control-at-2-2"></a>NIST 800-53 제어 AT-2(2)

#### <a name="security-awareness-training--insider-threat"></a>보안 인식 교육 | 내부자 위협

**AT-2(2)** 조직에는 잠재적인 내부자 위협 지표를 인식하고 보고하기 위한 보안 인식 교육이 포함되어 있습니다.

**책임:** `Customer Only`

|||
|---|---|
| **고객** | 고객의 엔터프라이즈 수준 보안 인식 교육 콘텐츠에 내부자 위협 인식이 포함될 수 있습니다. |
| **공급자(Microsoft Azure)** | 해당 없음 |


 ## <a name="nist-800-53-control-at-3a"></a>NIST 800-53 제어 AT-3.a

#### <a name="role-based-security-training"></a>역할 기반 보안 교육

**AT-3.a** 조직은 정보 시스템에 대한 액세스를 허가하거나 할당된 임무를 수행하기 전에 할당된 보안 역할 및 책임을 갖는 직원에게 역할 기반 보안 교육을 제공합니다.

**책임:** `Customer Only`

|||
|---|---|
| **고객** | 고객의 엔터프라이즈 수준 교육 프로그램은 역할 기반 보안 교육을 제공할 수 있습니다. |
| **공급자(Microsoft Azure)** | 해당 없음 |


 ## <a name="nist-800-53-control-at-3b"></a>NIST 800-53 제어 AT-3.b

#### <a name="role-based-security-training"></a>역할 기반 보안 교육

**AT-3.b** 조직은 정보 시스템 변경에 따라 요구되는 보안 역할 및 책임이 할당된 직원에게 역할 기반 보안 교육을 제공합니다.

**책임:** `Customer Only`

|||
|---|---|
| **고객** | 고객의 엔터프라이즈 수준 교육 프로그램은 필요 기반, 역할 기반 보안 교육을 제공할 수 있습니다. |
| **공급자(Microsoft Azure)** | 해당 없음 |


 ## <a name="nist-800-53-control-at-3c"></a>NIST 800-53 제어 AT-3.c

#### <a name="role-based-security-training"></a>역할 기반 보안 교육

**AT-3.c** 조직은 [할당: 조직이 정의한 빈도] 후에 할당된 보안 역할 및 책임을 갖는 직원에게 역할 기반 보안 교육을 제공합니다.

**책임:** `Customer Only`

|||
|---|---|
| **고객** | 고객의 엔터프라이즈 수준 교육 프로그램은 진행 중인 역할 기반 보안 교육을 제공할 수 있습니다. |
| **공급자(Microsoft Azure)** | 해당 없음 |


 ### <a name="nist-800-53-control-at-3-3"></a>NIST 800-53 제어 AT-3(3)

#### <a name="role-based-security-training--practical-exercises"></a>역할 기반 보안 교육 | 연습

**AT-3 (3)** 조직은 교육 목표를 강화하는 연습 과정을 보안 교육에 포함합니다.

**책임:** `Customer Only`

|||
|---|---|
| **고객** | 고객의 엔터프라이즈 수준 역할 기반 보안 교육에는 실무 연습 과정이 포함될 수 있습니다. |
| **공급자(Microsoft Azure)** | 해당 없음 |


 ### <a name="nist-800-53-control-at-3-4"></a>NIST 800-53 제어 AT-3(4)

#### <a name="role-based-security-training--suspicious-communications-and-anomalous-system-behavior"></a>역할 기반 보안 교육 | 의심스러운 통신 및 비정상적인 시스템 동작

**AT-3(4)** 조직은 직원에게 [할당: 조직이 정의한 악성 코드 지표]에 대한 교육을 제공하여 조직 정보 시스템의 의심스러운 통신 및 비정상적인 동작을 인식하도록 합니다.

**책임:** `Customer Only`

|||
|---|---|
| **고객** | 고객의 엔터프라이즈 수준 역할 기반 보안 교육에는 의심스러운 통신 및 비정상적인 동작에 대한 인식이 포함될 수 있습니다. |
| **공급자(Microsoft Azure)** | 해당 없음 |


 ## <a name="nist-800-53-control-at-4a"></a>NIST 800-53 제어 AT-4.a

#### <a name="security-training-records"></a>보안 교육 기록

**AT-4.a** 조직은 기본 보안 인식 교육 및 특정 정보 시스템 보안 교육을 포함하여 개별 정보 시스템 보안 교육 활동을 문서화하고 모니터링합니다.

**책임:** `Customer Only`

|||
|---|---|
| **고객** | 고객의 엔터프라이즈 수준 보안 인식 교육 프로그램에는 교육 활동의 문서화 및 모니터링이 포함될 수 있습니다. |
| **공급자(Microsoft Azure)** | 해당 없음 |


 ## <a name="nist-800-53-control-at-4b"></a>NIST 800-53 제어 AT-4.b

**책임:** `Customer Only`

|||
|---|---|
| **고객** | 고객의 엔터프라이즈 수준 보안 인식 교육 프로그램이 교육 기록 보존 요구를 충족할 수 있습니다. |
| **공급자(Microsoft Azure)** | 해당 없음 |

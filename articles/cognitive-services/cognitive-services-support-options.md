# <a name="azure-cognitive-services-support-and-help-options"></a>Azure Cognitive Services 지원 및 도움말 옵션

Azure Cognitive Services의 기능을 탐색 하는 것을 시작 하 고 있나요? 응용 프로그램에서 새로운 기능을 구현 하는 경우를 가정해 볼 수 있습니다. 또는 서비스를 사용한 후 개선 하는 방법에 대 한 제안이 있나요? 다음은 지원을 받고, 최신 상태를 유지 하 고, 피드백을 제공 하 고, Cognitive Services에 대 한 버그를 보고할 수 있는 옵션입니다.

## <a name="create-an-azure-support-request"></a>Azure 지원 요청 만들기

    <img alt='Azure support' src='https://docs.microsoft.com/media/logos/logo_azure.svg'>
</div>

지금 클라우드를 시작하려는 개발자든 비즈니스에 중요한 전략적 애플리케이션을 배포하려는 대규모 조직이든 관계없이 가장 적합한 [Azure 지원 옵션 및 플랜 선택](https://azure.microsoft.com/support/plans)의 범위를 탐색합니다. Azure 고객은 Azure Portal에서 지원 요청을 만들고 관리할 수 있습니다.

* [Azure Portal](https://ms.portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/overview)
* [미국 정부의 Azure Portal](https://portal.azure.us)

## <a name="stay-informed"></a>최신 소식 수신

Azure 블로그의 새 릴리스 또는 뉴스의 기능에 대 한 정보를 유지 하면 프로그래밍 오류, 서비스 버그 또는 Cognitive Services에서 아직 제공 되지 않는 기능 간의 차이점을 찾을 수 있습니다.

* [Azure 업데이트](https://azure.microsoft.com/updates/?category=ai-machine-learning&query=Azure%20Cognitive%20Services)의 제품 업데이트, 로드맵 및 공지에 대해 자세히 알아보세요.
* 문서 [의 새로운 기능](whats-new-docs.md) 에서 최근에 추가 되었거나 업데이트 된 Cognitive Services 문서를 확인 하세요.
* Cognitive Services에 대 한 뉴스는 [Azure 블로그에서](https://azure.microsoft.com/blog/topics/cognitive-services/)공유 됩니다.
* [Reddit](https://www.reddit.com/r/AZURE/search/?q=Cognitive%20Services&restrict_sr=1) 에 대 한 대화 Cognitive Services에 참여 합니다.

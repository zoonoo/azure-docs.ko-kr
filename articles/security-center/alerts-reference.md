# <a name="security-alerts---a-reference-guide"></a>보안 경고-참조 가이드

이 문서에서는 Azure Security Center의 위협 방지 모듈에서 볼 수 있는 보안 경고를 나열 합니다. 사용자 환경에 표시 되는 경고는 사용자 지정 구성 뿐만 아니라 보호 하는 리소스와 서비스에 따라 달라 집니다.

이러한 경고에 응답 하는 방법에 대 한 자세한 내용은 [Azure Security Center에서 보안 경고 관리 및 대응](security-center-managing-and-responding-alerts.md)을 참조 하세요.

경고 (및 권장 사항)를 내보내는 방법에 대 한 자세한 내용은 [보안 경고 및 권장 사항 내보내기 (미리 보기)](continuous-export.md)를 참조 하세요.

경고 테이블 아래에는 이러한 경고의 의도를 분류 하는 데 사용 되는 Azure Security Center kill 체인을 설명 하는 테이블이 있습니다. 

이 테이블에 포함 된 경고 유형:
* [Windows 컴퓨터에 대 한 경고](#alerts-windows)
* [Linux 컴퓨터에 대 한 경고](#alerts-linux)
* [Azure App Service에 대 한 경고](#alerts-azureappserv)
* [AKS 클러스터 수준에 대 한 경고](#alerts-akscluster)
* [컨테이너 호스트 수준에 대 한 경고](#alerts-containerhost)
* [SQL Database 및 SQL Data Warehouse에 대 한 경고](#alerts-sql-db-and-warehouse)
* [Azure Storage에 대 한 경고](#alerts-azurestorage)
* [Azure Cosmos DB에 대 한 경고](#alerts-azurecosmos)
* [Azure 네트워크 계층에 대 한 경고](#alerts-azurenetlayer)
* [Azure Resource Manager에 대 한 경고 (미리 보기)](#alerts-azureresourceman)
* [Azure Key Vault에 대 한 경고 (미리 보기)](#alerts-azurekv)
* [Azure DDoS Protection에 대 한 경고](#alerts-azureddos)

## <a name="intentions"></a>의도

공격 의도를 이해 하면 이벤트를 보다 쉽게 조사 하 고 보고할 수 있습니다. Azure Security Center 경고에는 이러한 작업에 도움이 되는 ' 의도 ' 필드가 포함 됩니다.

정찰에서 데이터 반출의 진행을 설명 하는 일련의 단계를 종종 "kill chain" 이라고 합니다. 

Security Center 지원 되는 kill 체인 의도는 [MITRE at&t & 접시™ 프레임 워크](https://attack.mitre.org/matrices/enterprise) 를 기반으로 하며 아래 표에 설명 되어 있습니다.

|Intent|Description|
|------|-------|
|**프로브**|검색은 악의적인 의도에 관계 없이 특정 리소스에 액세스 하려고 시도 하거나, 악용 하기 전에 정보를 수집 하기 위해 대상 시스템에 대 한 액세스를 시도할 수 있습니다. 일반적으로이 단계는 네트워크 외부에서 시작 하 여 대상 시스템을 검사 하 고 진입점을 식별 하는 시도로 검색 됩니다.|
|**개발**|악용은 공격자가 공격을 받는 리소스에 대 한 발판을 관리 하는 단계입니다. 이 단계는 계산 호스트 및 리소스 (예: 사용자 계정, 인증서 등)와 관련이 있습니다. 위협 행위자는이 단계 후에 리소스를 제어할 수 있는 경우가 많습니다.|
|**지속성**|지 속성은 해당 시스템에 대 한 위협 행위자를 지속적으로 확인 하는 시스템에 대 한 액세스, 작업 또는 구성 변경입니다. 위협 행위자는 시스템 다시 시작, 자격 증명 손실 또는 원격 액세스 도구를 다시 시작 해야 하는 기타 오류와 같은 중단을 통해 시스템에 대 한 액세스를 유지 관리 해야 하는 경우가 많습니다.|
|**PrivilegeEscalation**|권한 상승은 악의적 사용자가 시스템 또는 네트워크에 대 한 높은 수준의 사용 권한을 얻을 수 있도록 하는 작업의 결과입니다. 특정 도구나 작업을 수행 하려면 높은 수준의 권한이 필요 하며 작업 전체에서 많은 지점에 필요할 수 있습니다. 특정 시스템에 액세스 하거나 악의적 사용자 하는 데 필요한 특정 기능을 수행할 수 있는 권한이 있는 사용자 계정은 권한 에스컬레이션으로 간주 될 수도 있습니다.|
|**DefenseEvasion**|방어 evasion는 악의적 사용자가 검색을 피할 하거나 다른 방어를 방지 하는 데 사용할 수 있는 기술로 구성 됩니다. 이러한 작업은 특정 방어 또는 완화에 대 한 추가 이점을 제공 하는 다른 범주의 경우와 동일 합니다.|
|**CredentialAccess**|자격 증명 액세스는 엔터프라이즈 환경 내에서 사용 되는 시스템, 도메인 또는 서비스 자격 증명에 대 한 액세스 또는 제어를 초래 하는 기술을 나타냅니다. 악의적 사용자는 네트워크 내에서 사용 하기 위해 사용자나 관리자 계정 (로컬 시스템 관리자 또는 관리자 액세스 권한이 있는 도메인 사용자) 으로부터 합법적인 자격 증명을 얻으려고 시도할 수 있습니다. 네트워크 내에서 충분 한 액세스를 사용 하 여 악의적 사용자가 환경 내에서 나중에 사용할 계정을 만들 수 있습니다.|
|**검색**|검색은 악의적 사용자가 시스템 및 내부 네트워크에 대 한 정보를 얻을 수 있는 기술로 구성 됩니다. 새 시스템에 대 한 액세스 권한을 획득 하는 경우 해당 시스템에서 현재 제어 하 고 있는 것과 해당 시스템에서 작동 하는 혜택을 악의적 사용자 하 여 침입 중에 현재 목표 또는 전체 목표를 달성 해야 합니다. 운영 체제는이 손상 후 정보 수집 단계에서 도움이 되는 다양 한 기본 도구를 제공 합니다.|
|**LateralMovement**|측면 이동은 악의적 사용자가 네트워크를 통해 원격 시스템에 액세스 하 고 제어할 수 있는 기술로 구성 되어 있지만 반드시 원격 시스템에 도구를 실행할 수 있는 것은 아닙니다. 측면 이동 기법을 사용 하 여 악의적 사용자가 원격 액세스 도구와 같은 추가 도구 없이도 시스템에서 정보를 수집할 수 있습니다. 악의적 사용자는 도구의 원격 실행, 추가 시스템으로 피벗, 특정 정보 또는 파일에 대 한 액세스, 추가 자격 증명에 대 한 액세스 또는 효과 발생을 포함 하 여 다양 한 용도로 측면 이동을 사용할 수 있습니다.|
|**실행**|실행 기법은 로컬 또는 원격 시스템에서 악의적 사용자가 제어하는 코드를 실행하게 되는 기술을 나타냅니다. 이 기법은 네트워크의 원격 시스템에 대 한 액세스를 확장 하기 위해 측면 이동과 함께 사용 되는 경우가 많습니다.|
|**컬렉션**|컬렉션은 exfiltration 전의 대상 네트워크에서 중요 한 파일과 같은 정보를 식별 하 고 수집 하는 데 사용 되는 기술로 구성 됩니다. 또한이 범주는 악의적 사용자가 탈취에 대 한 정보를 찾을 수 있는 시스템 또는 네트워크의 위치를 포함 합니다.|
|**반출**|Exfiltration은 악의적 사용자가 대상 네트워크에서 파일 및 정보를 제거 하는 것을 돕는 기술 및 특성을 나타냅니다. 또한이 범주는 악의적 사용자가 탈취에 대 한 정보를 찾을 수 있는 시스템 또는 네트워크의 위치를 포함 합니다.|
|**CommandAndControl**|명령 및 제어 기법은 악의적 사용자가 대상 네트워크 내에서 제어 하는 시스템과 통신 하는 방법을 나타냅니다.|
|**영향**|영향 이벤트는 주로 시스템, 서비스 또는 네트워크의 가용성 또는 무결성을 직접 줄이기 위해 시도 합니다. 비즈니스 또는 운영 프로세스에 영향을 주는 데이터 조작을 포함 합니다. 이는 종종 랜 섬 웨어, 파손, 데이터 조작 등의 기법을 지칭 합니다.|

# <a name="role-based-access-control-rbac-for-azure-vmware-solution"></a>Azure VMware 솔루션에 대 한 RBAC (역할 기반 액세스 제어)

Azure VMware 솔루션에서 vCenter는 cloudadmin 이라는 기본 제공 로컬 사용자를 포함 하 고 기본 제공 CloudAdmin 역할에 할당 합니다. 로컬 cloudadmin 사용자는 AD에서 사용자를 설정 하는 데 사용 됩니다. 일반적으로 CloudAdmin 역할은 사설 클라우드에서 워크 로드를 만들고 관리 합니다. Azure VMware 솔루션에서 CloudAdmin 역할에는 다른 VMware 클라우드 솔루션과 다른 vCenter 권한이 있습니다.     

> [!NOTE]
> Azure VMware 솔루션은 현재 vCenter 또는 Azure VMware 솔루션 포털에서 사용자 지정 역할을 제공 하지 않습니다. 

VCenter 및 ESXi 온-프레미스 배포에서 관리자는 vCenter 계정에 액세스할 수 administrator@vsphere.local 있습니다. 또한 할당 된 추가 Active Directory (AD) 사용자/그룹이 있을 수 있습니다. 

Azure VMware 솔루션 배포에서 관리자는 관리자 사용자 계정에 액세스할 수 없습니다. 하지만 vCenter의 CloudAdmin 역할에 AD 사용자 및 그룹을 할당할 수 있습니다.  

사설 클라우드 사용자는 Microsoft에서 지원 및 관리 되는 특정 관리 구성 요소에 액세스할 수 없으며 구성할 수 없습니다. 예를 들어 클러스터, 호스트, 데이터 저장소 및 분산 가상 스위치가 있습니다.




## <a name="azure-vmware-solution-cloudadmin-role-on-vcenter"></a>VCenter의 Azure VMware 솔루션 CloudAdmin 역할

Azure VMware Solution 사설 클라우드 vCenter의 Azure VMware Solution CloudAdmin 역할에 부여 된 권한을 볼 수 있습니다.

1. SDDC vsphere 클라이언트에 로그인 하 고 **메뉴**  >  **관리**로 이동 합니다.
1. **Access Control**에서 **역할**을 선택 합니다.
1. 역할 목록에서 **Cloudadmin** 을 선택한 다음, **권한**을 선택 합니다.

# <a name="enable-the-preview-experience-to-administer-your-managed-domain-from-the-azure-portal"></a>미리 보기 환경을 사용하여 Azure Portal에서 관리되는 도메인 관리

## <a name="your-existing-managed-domain-is-not-shown-in-the-azure-portal-preview-experience"></a>기존의 관리되는 도메인은 Azure Portal(미리 보기) 환경에 표시되지 않습니다.
Azure 클래식 포털을 사용하여 만든 기존의 [Azure AD Domain Services](active-directory-ds-overview.md) 관리되는 도메인이 있습니다. Azure Portal로 이동하는 경우 Azure AD Domain Services 블레이드(미리 보기)에 이러한 기존의 관리되는 도메인이 표시되지 않습니다.

![기존의 관리되는 도메인이 Azure Portal에 표시되지 않습니다.](./media/getting-started/missing-domain-services-instance.png)

기존의 관리되는 도메인이 새 Azure Portal로 마이그레이션되지 않았기 때문입니다. 기존의 관리되는 도메인에 대한 새 Azure Portal(미리 보기) 환경을 사용하도록 설정하려면 Microsoft에 연락하세요.


## <a name="join-the-preview-for-the-azure-portal-experience-for-azure-ad-domain-services"></a>Azure AD Domain Services에 대한 Azure Portal 환경을 위해 미리 보기에 가입
[문의처](active-directory-ds-contact-us.md) 페이지의 지시에 따라 전자 메일을 보내세요. 기존의 관리되는 도메인에 대해 Azure Portal 미리 보기를 사용하도록 설정하는 작업은 Microsoft에서 권장되고 있습니다.

> [!NOTE]
> Azure AD Domain Services에 대한 새 Azure Portal 환경이 일반 공급(GA)되면 기존의 모든 관리되는 도메인이 자동으로 마이그레이션됩니다. 그러면 Azure Portal에서 기존의 관리되는 도메인을 관리할 수 있게 됩니다.

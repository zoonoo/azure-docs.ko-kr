# <a name="geofencing-and-geospatial-aggregation-scenarios-with-azure-stream-analytics"></a>Azure Stream Analytics 지 오 펜싱 및 지리 공간적 집계 시나리오

기본 제공 지리 공간적 함수를 사용 하 여 Azure Stream Analytics를 사용 하 여 지역 관리, 공동 공유, 연결 된 자동차, 자산 추적 등의 시나리오에 대 한 응용 프로그램을 빌드할 수 있습니다.

## <a name="geofencing"></a>지오펜싱

Azure Stream Analytics는 클라우드와 IoT Edge 런타임에서 짧은 대기 시간 실시간 지 오 펜싱 계산을 지원 합니다.

### <a name="geofencing-scenario"></a>지 오 펜싱 시나리오

제조 회사는 건물의 자산을 추적 해야 합니다. 모든 장치에 GPS를 장착 하 고 장치가 특정 영역을 벗어나면 알림을 수신 하려고 합니다.

이 예제에 사용 된 참조 데이터에는 각 건물에서 허용 되는 건물 및 장치에 대 한 지 오 펜스 정보가 있습니다. 참조 데이터는 정적 이거나 느리게 변경 될 수 있습니다. 이 시나리오에는 정적 참조 데이터가 사용 됩니다. 데이터 스트림은 장치 ID 및 현재 위치를 지속적으로 내보냅니다.

### <a name="define-geofences-in-reference-data"></a>참조 데이터에서 지역 구분 정의

GeoJSON 개체를 사용 하 여 지 오를 정의할 수 있습니다. 호환성 버전 1.2 이상을 사용 하는 작업의 경우 지역 구분를 WKT (잘 알려진 텍스트)를 사용 하 여 정의할 수도 있습니다 `NVARCHAR(MAX)` . WKT는 공간 데이터를 텍스트 형식으로 표시 하는 데 사용 되는 OGC (Open Geospatial Consortium) 표준입니다.

기본 제공 지리 공간적 함수는 정의 된 지역 구분를 사용 하 여 특정 지 오 펜스 polygon에 요소가 있는지 여부를 확인할 수 있습니다.

### <a name="generate-alerts-with-geofence"></a>지 오를 사용 하 여 경고 생성

장치는 이라는 스트림을 통해 1 분 마다 ID 및 위치를 내보낼 수 있습니다 `DeviceStreamInput` . 다음 표는 입력의 스트림입니다.

|DeviceID|GeoPosition|
|--------|-----------|
|“A”|"POINT (-122.13292341559497 47.636318374032726)"|
|“B”|"POINT (-122.13338475554553 47.63743531308874)"|
|C|"POINT (-122.13354001095752 47.63627622505007)"|

장치 스트림을 지 오 트 참조 데이터와 조인 하는 쿼리를 작성 하 고 장치가 허용 된 건물을 벗어날 때마다 경고를 생성할 수 있습니다.

```SQL
SELECT DeviceStreamInput.DeviceID, SiteReferenceInput.SiteID, SiteReferenceInput.SiteName 
INTO Output
FROM DeviceStreamInput 
JOIN SiteReferenceInput
ON st_within(DeviceStreamInput.GeoPosition, SiteReferenceInput.Geofence) = 0
WHERE DeviceStreamInput.DeviceID = SiteReferenceInput.AllowedDeviceID
```

다음 이미지는 지역 구분을 나타냅니다. 스트림 데이터 입력에 따라 장치가 어디에 있는지 확인할 수 있습니다.

![지역 구분 빌드](./media/geospatial-scenarios/building-geofences.png)

장치 "C"는 참조 데이터에 따라 허용 되지 않는 빌딩 ID 2 내에 있습니다. 이 장치는 건물 ID 3 내에 위치 해야 합니다. 이 작업을 실행 하면이 특정 위반에 대 한 경고가 생성 됩니다.

### <a name="site-with-multiple-allowed-devices"></a>허용 되는 장치가 여러 개인 사이트

사이트에서 여러 장치를 허용 하는 경우에는 장치 id의 배열을에 정의할 수 `AllowedDeviceID` 있으며, 절에서 User-Defined 함수를 사용 `WHERE` 하 여 스트림 장치 id가 해당 목록의 모든 장치 id와 일치 하는지 확인할 수 있습니다. 자세한 내용은 클라우드 작업에 대 한 [JAVASCRIPT udf](stream-analytics-javascript-user-defined-functions.md) 자습서 및 edge 작업에 대 한 [c # udf](stream-analytics-edge-csharp-udf.md) 자습서를 참조 하세요.

## <a name="geospatial-aggregation"></a>지리 공간적 집계

Azure Stream Analytics는 클라우드와 IoT Edge 런타임에서 짧은 대기 시간 내 실시간 지리 공간적 집계를 지원 합니다.

### <a name="geospatial-aggregation-scenario"></a>지리 공간적 집계 시나리오

Cab 회사는 현재 상위 수요를 발생 하는 도시를 중심으로 하는 cab 드라이버를 안내 하는 실시간 응용 프로그램을 구축 하려고 합니다.

회사는 도시의 논리적 지역을 참조 데이터로 저장 합니다. 각 지역은 영역 Id, 영역 이름 및 지 오를 기준으로 정의 됩니다.

다음 쿼리는 장치 스트림을 지 오 트 참조 데이터와 조인 하 고 15 분 마다 1 분 마다 지역별 요청 수를 계산 합니다.

```SQL
SELECT count(*) as NumberOfRequests, RegionsRefDataInput.RegionName 
FROM UserRequestStreamDataInput
JOIN RegionsRefDataInput 
ON st_within(UserRequestStreamDataInput.FromLocation, RegionsRefDataInput.Geofence) = 1
GROUP BY RegionsRefDataInput.RegionName, hoppingwindow(minute, 1, 15)
```

이 쿼리는 도시 내의 각 지역이 지난 15 분 동안 분 마다 요청 수를 출력 합니다. 이 정보는 Power BI 대시보드에서 쉽게 표시 하거나, Azure 기능과 같은 서비스와의 통합을 통해 SMS 문자 메시지로 모든 드라이버에 브로드캐스트 될 수 있습니다.

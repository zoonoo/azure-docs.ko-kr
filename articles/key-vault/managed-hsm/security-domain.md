# <a name="about-the-managed-hsm-security-domain"></a>관리 되는 HSM 보안 도메인 정보

관리 되는 HSM (HSM 보안 도메인)은 재해가 발생 한 경우 관리 되는 HSM을 복구 하는 데 필요한 핵심 자격 증명 집합입니다. 보안 도메인은 관리 되는 HSM 하드웨어 및 서비스 소프트웨어 enclaves 생성 되며 HSM의 "소유권"을 나타냅니다.

모든 관리 되는 HSM에는 작동 하는 보안 도메인이 있어야 합니다. 새 관리 되는 HSM을 요청 하면 프로 비전 되지만 보안 도메인을 다운로드할 때까지 활성화 되지 않습니다. 관리 되는 HSM이 프로 비전 되었지만 활성화 되지 않은 경우 두 가지 방법으로 활성화할 수 있습니다.
- 보안 도메인 다운로드는 기본 방법 이며, 다른 관리 되는 HSM과 함께 사용 하거나 총 재해 로부터 복구할 수 있도록 보안 도메인을 안전 하 게 저장할 수 있습니다.
- 이미 보유 하 고 있는 기존 보안 도메인을 업로드 하 여 동일한 보안 도메인을 공유 하는 여러 개의 관리 되는 HSM 인스턴스를 만들 수 있습니다.

## <a name="download-your-security-domain"></a>보안 도메인 다운로드

관리 되는 HSM이 프로 비전 되었지만 활성화 되지 않은 경우 보안 도메인을 다운로드 하면 모든 하드웨어의 전체 손실에서 복구 하는 데 필요한 핵심 자격 증명이 캡처됩니다. 보안 도메인을 다운로드 하려면 3 개 이상의 RSA 키 쌍을 만들고 보안 도메인 다운로드를 요청할 때 이러한 공개 키를 서비스로 보내야 합니다. 또한 나중에 보안 도메인을 해독 하는 데 필요한 최소 키 수 (쿼럼)를 지정 해야 합니다. 관리 되는 HSM은 보안 도메인을 초기화 하 고 [rivest-shamir-adleman의 비밀 공유 알고리즘](https://dl.acm.org/doi/10.1145/359168.359176)을 사용 하 여 제공 하는 공개 키로 암호화 합니다. 적어도 개인 키 쿼럼을 사용할 수 있는 경우에만 다운로드 하는 보안 도메인 blob의 암호를 해독할 수 있습니다. 보안 도메인의 보안을 보장 하려면 개인 키를 안전 하 게 유지 해야 합니다. 다운로드가 완료 되 면 관리 되는 HSM이 활성화 된 상태에서 사용할 수 있도록 준비 됩니다.  

> [!IMPORTANT]
> 전체 재해 복구를 위해서는 보안 도메인과이를 보호 하는 데 사용 된 개인 키의 쿼럼 및 전체 HSM 백업이 있어야 합니다. 보안 도메인을 분실 하거나 쿼럼을 손실 하는 RSA 키 (개인 키)가 부족 하 고 관리 되는 HSM의 실행 중인 인스턴스가 없는 경우 재해 복구를 수행할 수 없습니다.

## <a name="upload-a-security-domain"></a>보안 도메인 업로드

관리 되는 HSM이 프로 비전 되었지만 활성화 되지 않은 경우 보안 도메인 복구 프로세스를 시작할 수 있습니다. 관리 되는 HSM은 RSA 키 쌍을 생성 하 고 공개 키를 반환 합니다. 그런 다음 관리 되는 HSM에 보안 도메인을 업로드할 수 있습니다. 업로드 하기 전에 클라이언트 (Azure CLI 또는 PowerShell)는 보안 도메인을 다운로드 하는 동안 사용한 개인 키의 최소 쿼럼 수와 함께 제공 해야 합니다. 클라이언트는이 쿼럼을 사용 하 여 보안 도메인의 암호를 해독 하 고 복구를 요청할 때 다운로드 한 공개 키를 사용 하 여 다시 암호화 합니다. 업로드가 완료 되 면 관리 되는 HSM은 활성화 됨 상태가 됩니다.

## <a name="backup-and-restore-behavior"></a>백업 및 복원 동작

백업 (전체 백업 또는 단일 키 백업)은 원본으로 관리 되는 HSM (백업이 생성 된 위치)과 대상 관리 HSM (백업이 복원 되는 경우)이 동일한 보안 도메인을 공유 하는 경우에만 복원 될 수 있습니다. 이러한 방식으로 보안 도메인은 관리 되는 각 HSM의 암호화 경계도 정의 합니다.
